PROBLEM ANALYSIS
问题的本质
1
定位困难
Word 文档内容结构复杂，信息可能出现在正文段落、表格单元格、页眉页脚、文本框等不同位置，需要精准定位而非简单文本匹配。
2
格式保持
修改内容时必须保持原有格式样式——字体、字号、颜色、对齐方式等。格式混乱反而会增加后续工作量。
3
批量可靠性
处理大量文档时，必须确保每份文档都被正确处理，同时要有容错机制，避免单份文档问题导致整个流程中断。
4
可追溯性
修改完成后需要清晰记录，方便审核人员快速确认哪些内容被修改了，修改前后的对比是什么。

MODULE 01
内容定位模块
python-docx 将 Word 文档抽象为段落（Paragraph）和表格（Table）的集合。通过遍历这些元素，结合正则表达式和跨 run 匹配，可以精准定位到目标内容。
from docx import Document

doc = Document("report.docx")

# 遍历所有段落
for para in doc.paragraphs:
  if "关键词" in para.text:
    process(para)

# 遍历表格单元格
for table in doc.tables:
  for row in table.rows:
    for cell in row.cells:
      process(cell)

Document 文档对象

Paragraphs 段落集合
↓

精准匹配目标内容
